## Environment status

Alaya is currently in the **core stabilization phase** (see `docs/ROADMAP.md`). There is **no deployed environment** — no staging database, no production database, no tenant data. The repository is in pre-deployment core buildout.

Consequence: there are no production rows against which to run the preflight audit.

## Decision

- **Strict enforcement accepted as default.** Sprint 2 merged with full `ENTITY_TYPE_TIER_RANK` enforcement on every `part_of` write. Every caller (writer, panoramic, enrichment) catches `HierarchyViolationError` and logs-and-continues.
- **`ALAYA_PART_OF_STRICT` feature flag not implemented.** This was documented as a fallback path in the plan. The holistic review confirmed no call-sites reference the flag in this run. A backlog item is opened in `docs/backlog/BACKLOG.md` as `RUN6.1.FU.01` for a future follow-up IF a deployed environment later surfaces inverted rows that cannot be fixed in-place.

## Evidence that the audit script works

`scripts/audit_part_of_hierarchy.py` was exercised by the integration test `packages/core/tests/test_audit_part_of_script.py::test_audit_logic_directly` against an ephemeral testcontainer seeded with 2 valid + 2 tier-inverted `part_of` relations:

- Script returned count `2`.
- Exit code logic verified by the unit-level test `test_audit_script_exit_code_when_violations_found` — a non-zero count causes `SystemExit(1)`.
- Script is read-only (no `BEGIN TRANSACTION`, no mutation SQL).
- Sample projection includes `relation_id`, `source_slug`, `target_slug`, `source_name`, `target_name`, `created_at`, `extraction_run_id` per design §5.

When Alaya ships its first deployed environment (staging or production):

1. Set `ALAYA_DATABASE_URL` to the target DB.
2. Run `uv run python scripts/audit_part_of_hierarchy.py --sample-size 50`.
3. Append a new dated section to this file with the outcome.
4. If count > 0:
   - **Preferred:** Fix the inverted rows via a targeted data migration (not an Alembic migration — the migration stream stays at 007 per charter).
   - **Escape hatch:** Implement `ALAYA_PART_OF_STRICT=false` from the backlog item and temporarily warn instead of raising, then fix, then flip back.
